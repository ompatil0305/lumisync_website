# Brand, Copy, And SEO Standards

## Brand Direction

Lumisync should feel modern, calm, useful, and premium.

The visual identity should be university-neutral. Texas Tech can be the first supported university, but it must not become the global brand language.

## Visual Standards

Prefer:

- warm whites, soft grays, charcoal, black, and restrained accent colors
- generous spacing
- clear hierarchy
- real product screenshots
- restrained motion
- consistent buttons, cards, navigation, and section spacing

Avoid:

- generic AI SaaS gradients
- purple AI visual language
- excessive glow effects
- fake dashboard screenshots
- overdesigned hero cards
- university-specific colors in global tokens

## Copy Voice

The voice should be concise, confident, and specific.

Use plain language. Explain what Lumisync helps people do.

Better:

- "Find buildings, dining, parking, events, and faculty from one place."
- "Ask Lumi questions grounded in campus information."
- "Plan your day around what is open, nearby, and relevant."

Avoid:

- "Revolutionizing the student experience"
- "Unlock seamless AI-powered campus synergy"
- "The ultimate platform for everything"
- "Transforming the future of education"

## Claim Discipline

Every claim should be either:

- currently true in the product
- clearly marked as planned
- explicitly framed as a long-term vision

Do not imply live integrations, real-time data, indoor navigation, Canvas integration, or multi-university coverage unless the product actually supports them.

## SEO Standards

Every meaningful page should have:

- unique title
- useful meta description
- Open Graph image or fallback
- readable H1
- semantic heading structure
- crawlable text
- descriptive internal links
- content that matches search intent

Search traffic should be earned through useful pages, not keyword stuffing.

## Content Governance

When product capability changes, update the website copy in the same pull request or a linked follow-up.

When architecture changes affect public messaging, update both website docs and the relevant page copy.

When adding a new page, define its audience, purpose, primary CTA, SEO title, and success metric before implementation.
