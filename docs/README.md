# Lumisync Website Documentation

This folder is the source of truth for the Lumisync marketing website.

The website and the product application are part of the same company narrative, but they have different jobs. The application proves product value through utility. The website earns trust, explains the product, and converts the right audiences without overstating what exists.

## Current Website Stack

- Next.js 16
- React 19
- TypeScript
- Tailwind CSS 4
- Framer Motion
- lucide-react
- Radix UI primitives

## Documentation Map

- `website-strategy.md`: audience, positioning, route responsibilities, and conversion goals
- `brand-copy-seo.md`: brand voice, copy standards, SEO expectations, and content governance
- `quality-checklist.md`: launch and review checklist for website changes

## Relationship To The App Repo

The main Lumisync application docs live in the app repository. This website should stay aligned with the same product principles:

- platform before campus
- provider-first multi-university direction
- real product capability over fake marketing claims
- premium, neutral Lumisync identity
- accessibility and performance as launch requirements

Website contributors should not copy backend or app architecture rules blindly. Use this folder for marketing-site decisions and reference the app docs only when product architecture affects public messaging.

## Operating Principle

Every website change should answer one of these questions better:

- What is Lumisync?
- Who is it for?
- Why should students or universities trust it?
- What can the product actually do today?
- Where is the product going next?
